# Campaign Strategy Engine — Design Spec

**Date:** 2026-03-30
**Author:** Steven Junop + Claude
**Status:** Approved, ready for implementation
**Target:** April 6, 2026

## Problem

The Creative OS pipeline generates creatives but outputs no campaign strategy. The recruiting team gets 500+ ad images but no guidance on: which platforms to run them on, how to target audiences, what budget to allocate per ad set, when to kill or scale, or how to structure split tests. They still need a media buyer to turn the creative package into a live campaign.

## Solution

A Campaign Strategy Engine integrated into Stage 1 that transforms persona demographics + cultural research + budget into a ready-to-execute media plan. Each persona gets real targeting parameters (interests, behaviors, demographics) that map to ad platform targeting. Each country campaign gets a full Tier 1 media plan with ad sets, budget allocation, split test structure, kill/scale rules, and creative-to-ad-set mapping.

## Architecture

```
Intake Form (+ optional monthly budget)
  │
  ▼
Stage 1: Cultural Research (existing)
  │
  ▼
Stage 1: Persona Generation (ENRICHED with targeting_profile)
  │  - demographics: age_min/max, gender, education, occupation, income
  │  - interests: hyper (3+), hot (2-3), broad (3-5)
  │  - behaviors, psychographics, media_consumption
  │  - estimated_pool_size, expected_cpl_tier, budget_weight_pct
  │
  ▼
Stage 1: Campaign Strategy Generator (NEW)
  │  - Budget cascade: RFP → countries → personas → ad sets
  │  - Ad set optimization (min $10/day rule)
  │  - Split test plan (A/B, one variable)
  │  - Kill/scale rules per ad set
  │  - Tier 1 structure + progression rules
  │
  ▼
Stage 1: Campaign Strategy Evaluator (NEW, 7-dimension rubric)
  │  - Feedback loop: score < 0.80 → regenerate with issues → re-evaluate
  │
  ▼
Stage 1: Brief Generation (existing, now includes campaign_strategy)
  │
  ▼
Save to Neon (campaign_strategies table + enriched personas)
```

## Component 1: Persona Targeting Profile

Each persona gets a `targeting_profile` generated by the LLM based on cultural research + task type + persona demographics. This is a UNIVERSAL profile — platform-specific translation happens at ad set creation time.

### Schema

```json
{
  "archetype_key": "the_student",
  "name": "Amira",
  "age_range": "18-24",

  "targeting_profile": {
    "demographics": {
      "age_min": 18,
      "age_max": 24,
      "gender": "all",
      "education_level": "university",
      "occupation": "student",
      "income_bracket": "low",
      "languages": ["Arabic", "French"],
      "relationship_status": "single"
    },
    "interests": {
      "hyper": ["data annotation", "AI training data", "remote freelance work"],
      "hot": ["side hustle", "flexible work", "work from home"],
      "broad": ["technology", "part time jobs", "university life", "online earning"]
    },
    "behaviors": ["smartphone power user", "mobile payment user", "online course taker"],
    "psychographics": {
      "values": ["flexibility", "independence", "skill building"],
      "pain_points": ["class schedule conflicts", "low service job pay"],
      "media_consumption": ["TikTok heavy", "Instagram stories", "WhatsApp groups"]
    },
    "estimated_pool_size": "large",
    "expected_cpl_tier": "low",
    "budget_weight_pct": 45
  }
}
```

### Generation Rules

- `interests.hyper` — minimum 3, must be specific to the task type and persona (not generic like "technology")
- `interests.hot` — 2-3 stacked interests that combine to narrow the audience
- `interests.broad` — 3-5 broader interests to cover the full possible contributor base
- `budget_weight_pct` — LLM ranks personas by: estimated pool size in target region x inverse expected CPL. Highest opportunity persona gets highest weight. All persona weights must sum to 100.
- `estimated_pool_size` — "large", "medium", or "small" based on cultural research demographic data
- `expected_cpl_tier` — "low" (<$2), "medium" ($2-8), "high" (>$8) based on platform costs in the target region

### Platform Translation Layer

The universal targeting profile gets translated to platform-specific targeting at ad set creation. This is deterministic mapping, not LLM:

| Universal Field | Meta (FB/IG) | TikTok | LinkedIn | YouTube | X (Twitter) | Snapchat | Pinterest | WeChat |
|----------------|-------------|--------|----------|---------|-------------|----------|-----------|--------|
| `age_min/max` | Detailed targeting: age | Age targeting | Age targeting | Demographics: age | Age targeting | Age targeting | Age targeting | Age targeting |
| `education_level` | Education level | N/A | Degree | N/A | N/A | N/A | N/A | Education tag |
| `occupation` | Job title (limited) | N/A | Job title + function | N/A | N/A | N/A | N/A | N/A |
| `interests.hyper` | Interest targeting | Interest targeting | Skill + group | In-market audience | Keyword targeting | Snap Lifestyle Cat. | Interest + keyword | Interest + behavior |
| `interests.hot` | Stacked interests | Behavior + interest | Industry + seniority | Custom intent | Conversation topics | Stacked interests | Actalike targeting | Stacked tags |
| `interests.broad` | Broad demographics | Broad age/geo | Company size + function | Demographics | Interest categories | Demographics + geo | Broad interests | Demographics |
| `behaviors` | Behaviors targeting | Behavior targeting | N/A | N/A | N/A | N/A | N/A | Behavior tags |

## Component 2: Campaign Structure Engine

Each country campaign gets a fully specified Tier 1 media plan. The entire structure is LLM-generated (dynamic), not hardcoded. The framework (tiers, interest hierarchy, rules) is fixed; the content is generated.

### Campaign Strategy JSON

```json
{
  "campaign_strategy": {
    "tier": 1,
    "tier_description": "Interest-only cold traffic",
    "monthly_budget": 1000,
    "budget_mode": "fixed",
    "daily_budget_total": 33,
    "split_test": {
      "variable": "creative",
      "description": "Campaign A uses gradient overlay creatives, Campaign B uses split panel",
      "measurement": "Compare CPA after 100 leads per campaign"
    },
    "campaigns": [
      {
        "name": "Vega Morocco — Campaign A (Gradient)",
        "objective": "lead_generation",
        "optimization": "leads",
        "daily_budget": 16.50,
        "ad_sets": [
          {
            "name": "Hyper — Data Annotation Interest",
            "persona_key": "the_student",
            "targeting_type": "hyper",
            "interests": ["data annotation", "AI training data"],
            "demographics": {"age_min": 18, "age_max": 24, "gender": "all"},
            "placements": ["ig_feed", "ig_story", "facebook_feed"],
            "daily_budget": 5.50,
            "kill_threshold": 8.25,
            "kill_rule": "No leads after $8.25 spend — pause ad set",
            "scale_rule": "CPA below target after 10 leads — increase budget 20% every 3 days",
            "creative_assignment_rule": {"persona": "the_student", "hook_types": ["earnings", "curiosity"], "treatment": "gradient_overlay"},
            "creatives_assigned": []
          }
        ]
      }
    ],
    "progression_rules": {
      "next_tier": 2,
      "trigger": "250 tracked leads across all ad sets",
      "adds": ["1% Lead Lookalike audience ad set", "Retargeting — visitors exclude leads"],
      "estimated_timeline": "2-4 weeks at current budget"
    },
    "scaling_rules": {
      "winning_ad_set": "Increase daily budget by 20% every 3 days if CPA stable",
      "losing_ad_set": "Pause after kill_threshold with no leads",
      "winning_creative": "Duplicate to new ad set with fresh audience segment"
    },
    "deferred_markets": []
  }
}
```

### What Is Dynamic (LLM-Generated)

- Number of ad sets (budget math determines count)
- Interest keywords (cultural research + task type + persona)
- Demographics per ad set (from persona targeting_profile)
- Persona budget weights (LLM ranks by pool size x inverse CPL)
- Country budget weights (LLM ranks by market opportunity)
- Split test variable (LLM decides: creative OR copy)
- Creative assignment RULES per ad set (persona + hook type + treatment — actual creative IDs filled in post-Stage 4)
- Progression timeline estimate (budget x market size x CPL)
- Number of campaigns (budget-dependent, might be 1 if tight)
- Ad set names (generated from persona + targeting type + interest)
- Tier starting point (skip to Tier 2 if client has contact list)
- Platform-specific placements per ad set

### What Is Structural (Fixed Framework)

- Tiered progression model (Tier 1 through 5)
- Interest hierarchy concept (hyper → hot → broad)
- Kill/scale rule existence (every ad set MUST have one)
- Split test requirement (every launch MUST have one)
- Budget minimum checks (math is fixed, thresholds are dynamic)

## Component 3: Budget Cascade Rules

### When Budget Is Specified (fixed mode)

```
RFP monthly budget
  → Country split (weighted by market opportunity, LLM-determined)
    → Persona split within country (weighted by budget_weight_pct)
      → Ad set split within persona (even, subject to $10/day minimum)
```

### Budget Optimization Rules

**Rule 1: Minimum $10/day per ad set**
```
per_ad_set_daily = (monthly / 30) × country_weight × persona_weight / num_ad_sets

IF per_ad_set_daily < $10:
  Reduce ad sets. Cut priority (first to go → last to go):
    1. Broad interests (Ad Set 5) — least targeted
    2. Hot interests (Ad Set 4) — decent but not hyper
    3. Keep: Hyper interests (Ad Sets 1-3) — highest quality
  Minimum: 2 ad sets per persona

IF even 2 ad sets < $10/day:
  Flag persona as underfunded in brief
  Recommend: increase budget or exclude persona
```

**Rule 2: Minimum viable country budget**
```
Minimum viable = num_personas × 2 ad sets × $10/day × 30 days
Typical: 3 personas × 2 × $10 × 30 = $1,800/mo per country

IF country_budget < minimum_viable:
  Rank countries by: estimated_pool_size × inverse_cpl (biggest bang for buck)
  Cut lowest-ranked countries until remaining are above minimum
  Cut countries saved as deferred_markets with reactivation threshold
  Log: "Cut [country] — insufficient budget. Deferred to Phase 2."
```

**Rule 3: Auto-consolidation for multi-country campaigns**
```
IF total_budget / num_countries < minimum_viable for ALL countries:
  Reduce countries to: floor(total_budget / minimum_viable)
  Select top countries by market opportunity ranking
  Deferred countries get specific reactivation budget threshold
```

### When Budget Is Not Specified (ratio mode)

- Country split: percentage ratios weighted by market opportunity
- Persona split: percentage ratios weighted by pool size
- Ad set count: "3-5 depending on budget" (team decides)
- No dollar amounts, no minimum checks, no auto-consolidation
- Brief says: "Plug in your monthly budget. Multiply by ratios below."

## Component 4: Campaign Strategy Evaluator

7-dimension rubric. Each dimension scored 0.0-1.0. Weighted average must pass 0.80 threshold.

| # | Dimension | Weight | Checks |
|---|-----------|--------|--------|
| 1 | **Targeting specificity** | 20% | Hyper interests are actually specific (not just "technology"). At least 3 hyper interests per persona. Interests match the task type (audio recording task → "voice acting", "podcast", not "data entry"). |
| 2 | **Persona-targeting alignment** | 15% | Student persona targets 18-24, not 18-65. Parent targets 28-45. Demographics match persona archetype. Occupation matches. Income bracket is realistic. |
| 3 | **Budget math validity** | 20% | Every ad set >= $10/day (or auto-reduced with flag). No persona underfunded without explicit flag. Country splits sum to total. Persona weights sum to 100%. Kill thresholds calculated correctly (1.5x daily). |
| 4 | **Platform-channel fit** | 15% | No TikTok for 50+ professionals. No LinkedIn for 18yo students (unless specific market data supports it). Platform matches persona's media_consumption. Placements match channel strategy from brief. |
| 5 | **Split test structure** | 10% | Exactly ONE variable differs between Campaign A and B. Variable is clearly identified. Measurement criteria specified. Not zero variables (no test) and not multiple (untestable). |
| 6 | **Kill/scale rules present** | 10% | Every ad set has a numeric kill threshold. Every winning rule has a specific action with numbers ("20% increase every 3 days" not "increase budget"). No vague "monitor performance." |
| 7 | **Tier-appropriate structure** | 10% | Tier 1 = interest-only (no LLAs, no retargeting). LLAs only in Tier 2+. Retargeting only with enough traffic. Progression trigger is specific ("250 leads" not "enough data"). Starting tier matches available data (has buyer list → can start Tier 4). |

### Evaluation Flow

```
LLM generates campaign_strategy JSON
  → Evaluator scores 7 dimensions
  → IF score >= 0.80: PASS → save to Neon
  → IF score < 0.80: collect issues → feed back to LLM → regenerate
  → Max 3 attempts
  → IF all 3 fail: save best attempt with warnings, flag for human review
```

### Budget-mode-specific evaluation

When `budget_mode = "ratio"` (no dollar amount specified):
- Skip dimension 3 (budget math validity) — no numbers to validate
- Weight redistributed to other dimensions proportionally
- Skip auto-consolidation and minimum checks
- Still evaluate targeting, alignment, platform fit, split test, kill/scale rules, tier structure

## Component 5: Database Schema

### New Table: `campaign_strategies`

```sql
CREATE TABLE IF NOT EXISTS campaign_strategies (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  request_id       UUID REFERENCES intake_requests(id) ON DELETE CASCADE,
  country          TEXT NOT NULL,
  tier             INT DEFAULT 1,
  monthly_budget   NUMERIC,
  budget_mode      TEXT DEFAULT 'ratio' CHECK (budget_mode IN ('fixed', 'ratio')),
  strategy_data    JSONB NOT NULL,
  evaluation_score NUMERIC,
  evaluation_data  JSONB,
  evaluation_passed BOOLEAN,
  version          INT DEFAULT 1,
  created_at       TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_campaign_strategies_request ON campaign_strategies(request_id);
```

### New Columns on Existing Tables

```sql
-- Persona targeting data on actor profiles
ALTER TABLE actor_profiles ADD COLUMN IF NOT EXISTS targeting_profile JSONB;

-- Budget metadata on creative briefs
ALTER TABLE creative_briefs ADD COLUMN IF NOT EXISTS budget_data JSONB;

-- Ad set assignment on generated assets (maps creative → ad set)
ALTER TABLE generated_assets ADD COLUMN IF NOT EXISTS ad_set_assignment JSONB;
```

### Intake Request Form Data Addition

Add optional `monthly_budget` field to the intake form:
```json
{
  "form_data": {
    "monthly_budget": 5000,
    "...existing fields..."
  }
}
```

If `monthly_budget` is present and > 0: `budget_mode = "fixed"`, run all budget math.
If absent or 0: `budget_mode = "ratio"`, output percentages only.

## Cross-Platform Campaign Framework

The tiered campaign structure (from the MBA Lead Gen Strategy doc) is channel-agnostic. It applies to all 9 supported platforms:

| Platform | Hyper Interests | Hot Interests | Broad | LLA Equivalent | Retargeting |
|----------|----------------|---------------|-------|----------------|-------------|
| Meta (FB/IG) | Interest targeting | Stacked interests | Broad demographics | Lookalike audiences | Pixel visitors |
| TikTok | Interest targeting | Behavior + interest | Broad age/geo | Custom audience LLA | Pixel/engagement |
| LinkedIn | Job title + skill | Industry + seniority | Company size + function | Matched audience LLA | Website/engagement |
| YouTube | In-market + affinity | Custom intent | Demographics | Similar audiences | Remarketing lists |
| X (Twitter) | Keyword + follower LLA | Conversation topics | Interest categories | Tailored audience LLA | Website tag retarget |
| Snapchat | Snap Lifestyle Categories | Stacked interests | Demographics + geo | Snap Audience Match LLA | Pixel/engagement |
| Pinterest | Interest + keyword | Actalike targeting | Broad interests | Actalike audiences | Engagement retarget |
| WeChat | Interest + behavior | Stacked tags | Demographics | Mini Program LLA | Official Account followers |
| WhatsApp | Via Meta Ads Manager | Same as Meta | Same as Meta | Same as Meta | Same as Meta |

The LLM generates platform-appropriate interest keywords and targeting parameters for each ad set based on which platforms are in the channel strategy.
